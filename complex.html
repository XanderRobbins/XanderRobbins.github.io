Module 01

Propositional Logic

What is a Proposition?

A proposition is a declarative sentence that is either true or false, but not both. Whether the statement is obviously true or false does not determine whether it is a proposition — we only need to know that a true or false value can be established.

Interrogative and imperative sentences like "Did you try binary search?" or "Always comment your code" are NOT propositions — they can't be true or false.
✅ Proposition

"Washington D.C. is the capital of the USA." — Has a definite truth value (True).

✅ Proposition

"33 is a prime number." — Has a definite truth value (False: 33 = 3×11).

✅ Proposition

"Every even integer > 2 is the sum of two primes." — Has a truth value (Goldbach's Conjecture, unknown but valid proposition).

❌ Not a Proposition

"What time is it?" — Interrogative, cannot be true or false.

❌ Not a Proposition

"x + 1 = 3" — Contains a free variable; truth depends on x.

Conditional Statements — Multiple Forms

The conditional statement p → q ("if p then q") can be expressed many ways in English. All of the following mean the same thing:

"if p, then q"
"p implies q"
"p only if q"
"q if p"
"q whenever p"
"q is necessary for p"
"p is sufficient for q"
"q unless ¬p"

Converse, Inverse, Contrapositive

From an implication p → q, we can derive three related statements:

Module 02

Propositional Equivalencies

Tautologies, Contradictions, and Contingencies

✅ Tautology

A compound proposition that is always true regardless of truth assignments.

p ∨ ¬p ≡ T
❌ Contradiction

A compound proposition that is always false. Also called an unsatisfiable proposition.

p ∧ ¬p ≡ F
⚡ Contingency

Neither a tautology nor a contradiction — sometimes true, sometimes false.

p → q
A proposition is satisfiable if it is true under at least one truth assignment. Tautologies and contingencies are satisfiable. Contradictions are not.

Logical Equivalence Laws

Two propositions p and q are logically equivalent (written p ≡ q) if they have the same truth value under every truth assignment — i.e., p ↔ q is a tautology.

Module 04

Predicates & Quantifiers

When Propositional Logic Fails

Propositional logic can't capture statements like:

"All candy made with chocolate is delicious."
"M&M's are made with chocolate."
∴ M&M's are delicious.

We need predicate logic — which adds variables, predicates, and quantifiers.

Variables

x, y, z — the subjects of our statements. Like variables in programming, they represent elements from a domain.

Predicates

Properties that variables can have. P(x): "x is greater than 3". Denoted with capital letters: P(x), Q(x,y).

Domain (Universe U)

The set of all possible values for variables. Must be specified! E.g., integers, real numbers, students in class.

Propositional Functions

A propositional function P(x) becomes a proposition (with a truth value) when we substitute a specific value for x from the domain.

Interactive Examples — P(x) = "x > 0" with domain ℤ

P(−3) = False P(0) = False P(3) = True P(100) = True
Let R(x,y,z) denote "x + y = z" with domain ℤ. Find: R(2, −1, 5).
R(2, −1, 5): Is 2 + (−1) = 5? → 1 = 5? → False
Let R(x,y,z) denote "x + y = z". Find: R(3, 4, 7).
R(3, 4, 7): Is 3 + 4 = 7? → 7 = 7? → True

Quantifiers

Quantifiers turn propositional functions into propositions by specifying how many elements in the domain satisfy the predicate.

∀

Universal Quantifier — "For All"

∀x P(x) — P(x) is true for every x in the domain.

Think: ∀x P(x) ≡ P(x₁) ∧ P(x₂) ∧ ... ∧ P(xₙ)

True when: P(x) holds for every x in the domain
False when: There exists at least one x where P(x) is false (a counterexample)
∃

Existential Quantifier — "There Exists"

∃x P(x) — P(x) is true for at least one x in the domain.

Think: ∃x P(x) ≡ P(x₁) ∨ P(x₂) ∨ ... ∨ P(xₙ)

True when: There is at least one x in the domain where P(x) is true
False when: P(x) is false for every x in the domain

Quick Evaluation Practice

Let P(x) = "x² > 0" with domain ℤ. Is ∀x P(x) true?
False — P(0) = "0² > 0" = "0 > 0" = False. x = 0 is a counterexample!
Let P(x) = "x² > 0" with domain ℤ. Is ∃x P(x) true?
True — P(1) = "1² > 0" = "1 > 0" = True. x = 1 (and many others) satisfy P(x).
∃! is the uniqueness quantifier. ∃!x P(x) means P(x) is true for exactly one x. With P(x) = "2x = 4" over ℤ, is ∃!x P(x) true?
True — Only x = 2 satisfies 2x = 4. There is exactly one solution.

De Morgan's Laws for Quantifiers — Negation

Just like De Morgan's laws for propositions, there are rules for negating quantifiers:

¬∀x P(x)
≡
∃x ¬P(x)
Negating "for all" gives "there exists some x where it fails"
¬∃x P(x)
≡
∀x ¬P(x)
Negating "there exists" gives "it fails for all x"

Negation Practice

Negate: "There is an honest politician."
Original: ∃x H(x) [H(x) = "x is honest"]
Negation: ¬∃x H(x) ≡ ∀x ¬H(x)
In English: "No politician is honest" (every politician is dishonest).
Negate: "All Americans eat cheeseburgers."
Original: ∀x C(x) [C(x) = "x eats cheeseburgers"]
Negation: ¬∀x C(x) ≡ ∃x ¬C(x)
In English: "There is some American who does not eat cheeseburgers."

Module 05

Nested Quantifiers

Nested Quantifiers

We can use multiple quantifiers together. The order matters for mixed quantifiers (∀∃ ≠ ∃∀)!

"Every real number has an additive inverse."
∀x ∃y (x + y = 0)
∀x ∀y P(x,y)

For all x and for all y, P(x,y) is true. Symmetric — order doesn't matter here.

∃x ∃y P(x,y)

There exist some x and y such that P(x,y) is true. Symmetric — order doesn't matter here.

∀x ∃y P(x,y)

For every x, there exists some y (possibly different for each x!) such that P(x,y) holds.

∃y ∀x P(x,y)

There is a SINGLE y that works for ALL x simultaneously. Much stronger than ∀x ∃y!

⚠️ Critical distinction: ∀x ∃y P(x,y) means each x can have its own y. But ∃y ∀x P(x,y) requires one fixed y that works for every x. These are usually NOT equivalent!

Truth Value Evaluation

P(x,y) = "xy = yx" over domain ℝ

Is ∀x ∀y P(x,y) true?
True — Multiplication is commutative over the reals: xy = yx for all real x and y.
Let Q(x,y) = "x + y = 5" over domain ℝ. Is ∀x ∃y Q(x,y) true?
True — For any x, choose y = 5 − x. Then x + (5 − x) = 5. Each x gets its own y.
Is ∃y ∀x Q(x,y) true (where Q(x,y) = "x + y = 5")?
False — This asks if there is ONE fixed y that makes x + y = 5 for ALL x simultaneously. But if y = 3, it only works for x = 2. No single y satisfies all x at once.
Let P(x,y) = "x·y = 0" over domain ℝ. Evaluate ∀x∀y P(x,y), ∀x∃y P(x,y), ∃x∀y P(x,y), ∃x∃y P(x,y).
∀x∀y P(x,y): False — 2·3 = 6 ≠ 0
∀x∃y P(x,y): True — For any x, choose y = 0. Then x·0 = 0 ✓
∃x∀y P(x,y): True — Choose x = 0. Then 0·y = 0 for all y ✓
∃x∃y P(x,y): True — x = 0, y = 0 works ✓

Negating Nested Quantifiers

Push the negation inside, flipping quantifiers each time you pass one:

¬∀x ∃y P(x,y)
≡
∃x ∀y ¬P(x,y)
Flip ∀→∃, then ∃→∀, negate the predicate
¬∃x ∀y P(x,y)
≡
∀x ∃y ¬P(x,y)
Flip ∃→∀, then ∀→∃, negate the predicate

Negation Practice

Find the negation of ∀x ∃y P(x,y) where P(x,y) = "(x = −y)" over ℝ. Express in English.
¬∀x ∃y P(x,y) ≡ ∃x ∀y ¬P(x,y) ≡ ∃x ∀y (x ≠ −y)

In English: "There exists a real number x such that for all real y, x ≠ −y."
(The original says every real number has an additive inverse; the negation says some real number does not.)
Translate: "There is a man that has taken a flight on every airline in the world." Let M(x): "x is a man", F(x,y): "x has taken flight y", A(y,z): "y operates on airline z".
∃m ∀a ∃f (M(m) ∧ F(m,f) ∧ A(f,a))
"There exists a man m such that for every airline a, there exists a flight f taken by m on airline a."

Rules of Inference

Arguments and Valid Arguments

An argument is a sequence of propositions (premises) that lead to a conclusion. An argument is valid if whenever all premises are true, the conclusion must also be true.

Example:
If it is raining, I will need an umbrella. [Premise 1]
It is raining. [Premise 2]
∴ I will need an umbrella. [Conclusion]

Resolution

Universal Instantiation (UI)

∀x P(x)
P(c)
Example:
"Every dog has 4 legs." (∀x D(x) → L(x))
"Oliver is a dog." (D(Oliver))
∴ UI gives us: D(Oliver) → L(Oliver)

If something is true for ALL elements, it is true for any specific element c.

Universal Generalization (UG)

P(c) for arbitrary c
∀x P(x)
Example:
If you can prove P(c) for an arbitrary (unspecified) c in the domain, you can conclude it holds for all x.

Used in math proofs where "let c be an arbitrary element."

Existential Instantiation (EI)

∃x P(x)
P(c) for some c
Example:
"Someone in class didn't read the book." (∃x ¬B(x))
We can give that person a name: call them c.
Then: ¬B(c) for this particular (but unspecified) c.

The c must be a NEW constant not used before.
CLICK FOR EXAMPLE

Existential Generalization (EG)

P(c) for some element c
∃x P(x)
Example:
"Alice has taken Discrete Math." P(Alice)
∴ "There exists someone who has taken Discrete Math." ∃x P(x)

If a specific element satisfies P, then there exists at least one element satisfying P.

Universal Modus Ponens

∀x (P(x) → Q(x))
P(a) — for particular a
Q(a)
Example:
"Every dog has 4 legs." ∀x (D(x) → L(x))
"Oliver is a dog." D(Oliver)
∴ "Oliver has 4 legs." L(Oliver)
